Javascript: Novice to Ninja

Chapter 11

~ Functions in JavaScript are first-class functions. This means functions can be passed as arguements to other functions, assigning them to variables, or storing them in data structures.

~ Once a temporary variable is defined, it is impossible to remove from scope.

~ Promise objects represent the eventual completion of an asynchronous operation and it's resulting value. It is a proxy value that might not be known when the promise is created. Those asynchronous functions are callbacks.

~ There are three Promise states: Pending, neither fulfilled nor rejected. Fulfilled, the operation was completed successfully. Rejected, the operation failed.

~ The process of turing a single function into a series of functions is called currying. It is the partial application of the functions.

Chapter 13

~ Fetch API is the standard to request/send asynchronous data. It provides fetch() (a global method) that has the arguement of the URL of the wanted resource. This fetch() returns a promise, the response returned from the URL.

~ The Same-Origin Policy is a security concept. It allows a web page to access data from a second web page if they both have the same origin. Otherwise it will block all requests from another page if it has a different origin (domain) from the page making the request.
