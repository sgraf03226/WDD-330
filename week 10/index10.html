Validating Forms

~ The purpose of client side form validation is to make sure that any data that is submitted through a form online will meet the necessary server requirements. It is important to note though, that this should not be the only form of security. However, it does provide some protection for the company's servers as well as the end user.

~ With form validation you can make certain parts required to fill out, create a min/max length of characters, or request a type (emails, phone numbers, etc.)

Using Fetch

~ A fetch statement requests data from a website through a link. The response is converted into a JSON format that can be modified through JS. The data is then logged into the console for the developer to view. The GET request unfortunately does not offer any security. If you would like to add security, use the POST method instead of the GET.

~ Example of Fetch: fetch('http://example.com/games.json') .then(response => response.json()) .then(data => console.log(data));

~ Example of POST: fetch('https://example.com', { credentials: 'include' });
